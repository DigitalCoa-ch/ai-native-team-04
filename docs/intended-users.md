# Intended Users

This system is designed for construction professionals who need real-time visibility into complex projects and early detection of coordination problems.

---

## User Types

| User Type | Primary Role | Key Need |
|---|---|---|
| **[Site Managers](site-managers.md)** | Senior on-site leaders | Holistic project visibility, critical alert prioritization |
| **[Project Supervisors](project-supervisors.md)** | Day-to-day coordinators | Cross-team coordination, progress tracking |
| **[Construction Foremen](construction-foremen.md)** | Trade team leaders | Simplified updates, early blocker warnings |
| **[Engineering Coordinators](engineering-coordinators.md)** | Technical specialists | Design alignment, clash detection |

---

## Quick Summary

### Site Managers
The executive view. Need to know immediately when critical issues arise and have confidence the system is catching problems before they become expensive. Primary interface: **dashboard + critical alerts**.

### Project Supervisors
The coordination hub. juggling updates from multiple foremen and need to spot coordination conflicts across trades. Primary interface: **unified progress view + cross-team alerts**.

### Construction Foremen
The front line. Don't want extra paperwork—need a fast way to report progress and get warned about blockers. Primary interface: **mobile updates + alert notifications**.

### Engineering Coordinators
The technical authority. Need the system to surface when field reality diverges from design intent, and to track design changes through to execution. Primary interface: **design comparison tools + clash detection**.

---

## Shared Principles

All users benefit from:
- **Reduced manual data gathering** — the system connects fragmented updates automatically
- **Earlier problem detection** — catching issues before they cause delays or safety risks
- **Clear accountability** — every alert has an owner and a paper trail
- **Human-in-the-loop validation** — the AI recommends; humans decide on critical actions
